# Hospital Readmission Predictor

An end-to-end machine learning pipeline for predicting **hospital readmission risk at discharge**, built on the checked-in `hospital_readmissions.csv` dataset (25,000 diabetic patient records).

---

## Project Overview

Hospital readmissions are costly, often preventable, and strongly associated with worse patient outcomes. A well-calibrated risk score at discharge allows care-management teams to prioritise follow-up resources — phone calls, home visits, medication reconciliation — for the patients most likely to return.

This project demonstrates a complete ML pipeline in a healthcare context:

| Stage | Description |
|---|---|
| **EDA** | Distributions, class balance, utilisation patterns, diagnosis breakdown |
| **Cleaning** | Target mapping, imputation, duplicate removal, schema validation |
| **Feature Engineering** | Age ordinal encoding, utilisation flags, specialty known flag, interaction terms |
| **Modelling** | Logistic Regression, Random Forest, Gradient Boosting (sklearn HGBC) |
| **Calibration** | Post-hoc sigmoid/isotonic calibration with held-out Brier evaluation |
| **Threshold Optimisation** | Recall-constrained threshold selection (recall ≥ 0.80) |
| **Interpretability** | SHAP global + local explanations on the full calibrated model |
| **Error Analysis** | TP/TN/FP/FN subgroup profiling |

## Dataset

| Property | Value |
|---|---|
| **File** | `data/raw/hospital_readmissions.csv` |
| **Rows** | 25,000 |
| **Columns** | 17 |
| **Target** | `readmitted` — `yes` → 1, `no` → 0 |
| **Positive rate** | 47.0% (nearly balanced) |
| **Source** | Diabetic patient records; detailed provenance is not documented in this repo |

| Column | Type | Notes |
|---|---|---|
| `age` | Categorical | Bracketed ranges: `[40-50)`, ..., `[90-100)` |
| `time_in_hospital` | Integer | Days admitted (1–14) |
| `n_lab_procedures` | Integer | Lab procedures during stay |
| `n_procedures` | Integer | Procedures during stay |
| `n_medications` | Integer | Distinct medications prescribed |
| `n_outpatient` | Integer | Prior outpatient visits |
| `n_inpatient` | Integer | Prior inpatient admissions |
| `n_emergency` | Integer | Prior emergency visits |
| `medical_specialty` | Categorical | 49.5% of values are `"Missing"` (valid category) |
| `diag_1` / `diag_2` / `diag_3` | Categorical | Diagnosis codes (8 categories each) |
| `glucose_test` | Categorical | `no` / `normal` / `high` |
| `A1Ctest` | Categorical | `no` / `normal` / `high` |
| `change` | Categorical | `yes` / `no` — medication change made |
| `diabetes_med` | Categorical | `yes` / `no` — diabetes medication prescribed |

## Quickstart

### 1. Set up the environment

```bash
git clone <repo-url>
cd hospital-readmission-risk
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Verify the dataset is in place

```
data/raw/hospital_readmissions.csv
```

### 3. Run notebooks in order

```bash
jupyter notebook notebooks/01_eda.ipynb
jupyter notebook notebooks/02_modeling_baseline.ipynb
jupyter notebook notebooks/03_model_tuning.ipynb
jupyter notebook notebooks/04_model_interpretation_and_error_analysis.ipynb
```

### 4. Run the pipeline programmatically

```python
from src.utils import load_config, set_seed
from src.data_preparation import load_raw_data, clean_data, encode_features
from src.feature_engineering import create_features

config = load_config()
set_seed(config['random_seed'])

df_raw      = load_raw_data(config['paths']['raw_data'])
df_clean    = clean_data(df_raw, config)
df_features = create_features(df_clean, config)
df_encoded  = encode_features(df_features, config)
```

## Pipeline Stages

### Stage 1 — EDA (`notebooks/01_eda.ipynb`)
- Shape, dtypes, summary statistics
- Class balance (≈ 47/53 — no collapse risk)
- Age bracket distribution, prior utilisation distributions
- `medical_specialty` missingness (49.5% "Missing")
- Correlation matrix, target correlations

### Stage 2 — Data Preparation (`src/data_preparation.py`)
- `load_raw_data(path)` — CSV ingestion with `FileNotFoundError`
- `clean_data(df, config)` — target mapping (`yes`→1, `no`→0), dedup, imputation, schema validation (raises on missing required columns)
- `encode_features(df, config)` — OHE with `drop_first=True`, post-encoding validation

### Stage 3 — Feature Engineering (`src/feature_engineering.py`)
- `age_ordinal` — ordered integer from bracket strings (replaces `age`)
- `any_n_inpatient`, `any_n_emergency` — binary prior utilisation flags
- `total_prior_utilization` — sum of all prior visit counts
- `specialty_known` — binary: `medical_specialty != "Missing"`
- Interaction terms: `n_inpatient × time_in_hospital`, `n_medications × time_in_hospital`

### Stage 4 — Modelling (`src/modeling.py`)
- Stratified 60/20/20 train/val/test split
- Baselines: Logistic Regression (in `StandardScaler` Pipeline), Random Forest, `HistGradientBoostingClassifier`
- 5-fold `StratifiedKFold` CV; primary metric: ROC-AUC
- `RandomizedSearchCV` hyperparameter tuning
- Post-hoc calibration (sigmoid / isotonic) with held-out Brier evaluation
- Recall-constrained threshold selection: find max-precision threshold where recall ≥ 0.80
- Saves `models/best_baseline_model.pkl` and `models/best_tuned_model.pkl`

### Stage 5 — Interpretability (`notebooks/04_model_interpretation_and_error_analysis.ipynb`)
- SHAP via model-agnostic `shap.Explainer` on the full calibrated model
- Feature values in plots show real (unscaled) values
- SHAP beeswarm (global importance), bar chart, dependence plots, waterfall examples
- Error group profiling: TP / TN / FP / FN subgroup statistics

## Configuration

All runtime parameters live in `config/config.yaml`:

| Key | Description |
|---|---|
| `paths.*` | File paths for raw data, processed data, model output |
| `random_seed` | Global seed for reproducibility |
| `data.target_column` | `"readmitted"` |
| `data.target_positive_value` | `"yes"` (mapped to 1) |
| `data.required_columns` | Schema declaration — `_validate()` raises if missing |
| `features.age_brackets` | Bracket → ordinal integer mapping |
| `features.prior_utilization_cols` | Columns to create binary flags for |
| `features.specialty_missing_value` | Sentinel value for unknown specialty |
| `model.*` | Hyperparameter defaults for each model type |
| `model.threshold.optimize_by` | `"recall"` — find max-precision at recall ≥ 0.80 |
